Filtered Backbone Collections

  • View On GitHub

This project is maintained by bjeanes

About

This Backbone plugin allows you to create sub-collections of any Backbone collection (or extended derivative) based on a filter function.

Sub-collections are bound to their parents and are kept up-to-date with changes. This means that you can use the sub-collection as live windows over a master collection of models, binding these windows to views as would you any other Backbone collection.

Furthermore, sub-collections inherit custom fields/methods that you add when extending raw Backbone collections. This means a sub-collection can be a completely transparent stand-in for the full parent collection.

Demo

Given a list of items:

var items = new Items([
    {shape: "square", color: "red"},
    {shape: "circle", color: "blue"},
    {shape: "square", color: "green"},
    {shape: "triangle", color: "red"},
    {shape: "rectangle", color: "purple"},
    {shape: "square", color: "red"}
]);

new ListView({el: $("#demo ul.all"), collection: items});

We can filter them by some arbitrary criteria (in this case, "squares"):

var squares = items.subset(function(item) {
    return item.get("shape") == "square";
});

new ListView({
    el: $("#demo ul.squares"),
    collection: squares
});

Then, for no good reason at all, we can filter that sub-collection even further (say, by "red"):

var redSquares = squares.subset(function(item) {
    return item.get("color") == "red";
});

new ListView({
    el: $("#demo ul.red-squares"),
    collection: redSquares
});
